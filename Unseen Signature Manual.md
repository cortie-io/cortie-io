# 😈 Deamon Project ID Manual: Unseen Signature System

## 1. 개요: ID, 기술적 선언이 되다

Deamon의 프로젝트 ID는 단순한 식별 코드가 아닌, **'보이지 않는 연결(Unseen Connection)'** 철학을 반영하는 기술적 선언입니다. 이 시스템은 **'무결점의 신뢰(Reliability)'**와 **'책임 있는 정교함(Precision)'**을 코드에 새기기 위해 설계되었습니다.

### ID 구조 공식 (총 14~16자리)

$$[ID] = [P] \ [YYMM] \ [EAWDC] \ [X] \ [Project Code] \ [Checksum]$$

---

## 2. ID 구성 요소 상세 매뉴얼

### A. Prefix (P) - 1자리 (필수)

Deamon의 **'선의를 위한 기능(Function for Good)'** 철학을 반영하여 프로젝트의 궁극적인 목표를 분류합니다.

| 코드 | 유형 | 의미 | 예시 프로젝트 |
| :---: | :--- | :--- | :--- |
| **G** | **Good** | 인류의 더 나은 삶, 지속 가능한 사회를 위한 공익/구조 솔루션. | 재난 구조 로봇, 에너지 효율 최적화 시스템. |
| **S** | **Solution** | 일반적인 고객 맞춤형 혁신 솔루션 및 상업 서비스. | 스마트 팩토리, 일반 B2B/B2C 서비스. |
| **R** | **Research** | 내부 R&D, 순수 기술 실험, 창의적/예술적 표현을 위한 탐구. | 호러 TV 실험, AI 상호작용 방식 연구. |

### B. Time Stamp (YYMM) - 4자리 (필수)

프로젝트가 공식적으로 시작되거나 등록된 연도와 월을 표기합니다. (예: 25년 10월 → **2510**)

### C. Fusion Type (EAWDC) - 5자리 (필수)

Deamon의 5가지 핵심 아키텍처 중 프로젝트에 사용된 기술 영역을 나타내어 **'Unseen Connection'**의 깊이를 표시합니다. 사용 시 해당 코드를, 미사용 시 **0 (제로)**를 입력합니다.

| 위치 | 코드 | 기술 영역 | 사용 시 입력 | 미사용 시 입력 |
| :---: | :---: | :--- | :---: | :---: |
| 1 | E | Embedded (제어, 펌웨어, 센서) | **E** | **0** |
| 2 | A | AI (인지, 지능, 분석, 모델링) | **A** | **0** |
| 3 | W | Web (인터페이스, Front-end, UX) | **W** | **0** |
| 4 | D | Data (DB, 저장, 파이프라인, 관리) | **D** | **0** |
| 5 | C | Communication (네트워크, API, 백엔드 서버) | **C** | **0** |

**예시:** AI 분석 백엔드만 있는 경우: **0A0DC**

### D. Specialty Code (X) - 1자리 (필수/특수)

EAWDC로 포괄하기 어려운 특수 기술 또는 미래 기술을 위한 유보 영역입니다.

| 코드 | 유형 | 의미 |
| :---: | :--- | :--- |
| **0** | None | 특수 기술 미사용 (대부분의 프로젝트) |
| **B** | Blockchain | 분산 원장 기술, NFT, 스마트 컨트랙트 |
| **Q** | Quantum | 양자 컴퓨팅 또는 양자 보안 기술 |
| **H** | Hardware | 특수 하드웨어/ASIC 설계 |

### E. Project Code - 3자리 (필수)

각 Prefix (G, S, R) 내에서 순차적으로 부여되는 고유 시퀀스 번호입니다. **001**부터 시작합니다.

### F. Checksum - 1~3자리 (필수)

**'무결점의 신뢰'**를 보장하기 위한 검증 코드입니다. ID의 오타나 오류를 검증하며, **Modulus 9 방식** 등을 통해 산출됩니다.

---

## 3. ID 생성 및 검증 절차

### 단계 1: 프로젝트 성격 및 기술 확정
- Prefix (P): G, S, R 중 하나 선택
- Fusion Type (EAWDCX): 사용 기술에 따라 6자리 코드 확정
- YYMM / Project Code: 확정된 날짜 및 순번 입력

### 단계 2: Checksum 산출
1. ID 전체 문자를 미리 정의된 규칙에 따라 숫자로 변환합니다. (G=7, S=19, E=5, A=1, W=23, D=4, C=3)
2. 변환된 모든 숫자를 합산하여 **Total Sum**을 구합니다.
3. Total Sum을 **9로 나눈 나머지**를 Checksum 값으로 확정합니다.

### 단계 3: 최종 ID 완성
산출된 Checksum 값을 최종적으로 ID 끝에 추가합니다.

| 예시: 재난 구조 로봇 | **G 2406 EAWDC 0 005 8** |
| :--- | :--- |
| **풀이** | 2024년 6월 시작된 **G**ood 프로젝트. 모든 핵심 기술(EAWDC)을 사용했으며, 특수 기술(X)은 없고, 5번째 순번. 검증 코드 **8**. |

---

## 4. ID 유지보수 및 관리

- **배타적 순번:** G, S, R 각각의 Prefix는 별도의 순번(Project Code)을 관리해야 합니다.
- **Fusion Type 수정:** 프로젝트 진행 중 기술 스택이 변경되어 Fusion Type이 달라질 경우, ID는 **가장 최신 기술 스택**을 반영하여 재발급되어야 합니다.
- **Checksum 유효성:** ID를 데이터베이스나 시스템에 입력할 때, 반드시 Checksum을 통해 유효성을 검증하여 **데이터 무결점**을 유지해야 합니다.
